\section{Monitoring Interfaces}
\label{sec:MonitoringInterfaces}

The monitoring subsystem exposes three interfaces over the same Kafka Streams state:
\begin{itemize}
\item an HTTP query service,
\item a command-line client,
\item a browser dashboard.
\end{itemize}

\subsection{HTTP service}

Entry point: \texttt{ProcessMonitoringApp}. Query layer:
\texttt{ProcessMonitoringQueryService} and \texttt{ProcessMonitoringHttpServer}.

Endpoints:
\begin{itemize}
\item \texttt{/health} --- Kafka Streams readiness,
\item \texttt{/instances/\{processInstanceId\}} --- latest instance view,
\item \texttt{/processes/\{processId\}/counts} --- process-specific state counts,
\item \texttt{/processes/\{processId\}/latency} --- activity latency summaries,
\item \texttt{/processes/\{processId\}/trends} --- bucketed start/completion/failure history,
\item \texttt{/counts} --- global state-count view,
\item \texttt{/stuck} --- stuck-instance detection,
\item \texttt{/dashboard} --- browser dashboard.
\end{itemize}

\subsection{CLI client}

\texttt{ProcessMonitoringClient} provides terminal access for development and
exploration:

\begin{itemize}
\item \texttt{health},
\item \texttt{instance <processInstanceId>},
\item \texttt{counts [processId]},
\item \texttt{latency <processId>},
\item \texttt{stuck [processId] [olderThanSeconds]}.
\end{itemize}

\subsection{Dashboard}

Served from \texttt{/dashboard}. Polls the same HTTP endpoints and presents
health, counts, latency, stuck instances, and instance details. Built as a
standalone Svelte application (\filepath{monitoring-ui/}) that compiles to
static assets served by the Quarkus backend.

\subsection{How the interfaces fit together}

All three interfaces consume the same materialized state:
\begin{enumerate}
\item events are published to \texttt{process-events},
\item Kafka Streams materializes state and counts,
\item the HTTP layer exposes that materialized state,
\item the CLI and dashboard consume the same HTTP API.
\end{enumerate}

The operator workflow is:
\begin{itemize}
\item publish a scenario or task outcome,
\item watch \texttt{process-events},
\item query one instance or inspect counts and latency,
\item use the dashboard for a broader view.
\end{itemize}
