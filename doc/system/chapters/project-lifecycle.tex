\section{Generated Project Lifecycle}
\label{sec:ProjectLifecycle}

Generated projects are used iteratively, not as one-off code dumps:

\begin{enumerate}
\item model a BPMN process,
\item scaffold a project from that model,
\item inspect generated topics, classes and helper scripts,
\item run demo flows or manual probes,
\item observe the lifecycle stream and monitoring views,
\item revise the BPMN model and regenerate.
\end{enumerate}

\subsection{Two modes of use}

\begin{itemize}
\item as a learning scaffold, where the value is seeing how BPMN maps onto Kafka,
\item as a starting point for a real implementation, where the generated code becomes a
base that is extended, completed and hardened.
\end{itemize}

The BPMN model remains the primary design artifact. Generated output is readable, runnable
and observable by design.
