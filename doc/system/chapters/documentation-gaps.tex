\section{Documentation Gaps Addressed}
\label{sec:DocumentationGaps}

This manual originally described the generator, runtime, monitoring path and sample models,
but it still lacked several system-documentation layers that matter for engineering use.
The main gaps were:
\begin{enumerate}
\item architecture decisions and the reasoning behind the current design,
\item an explicit statement of BPMN support limits and semantic boundaries,
\item a testing and verification summary,
\item configuration and local operations guidance,
\item a generated-project lifecycle from modeling through observation,
\item a shared glossary for BPMN, Kafka and Durga-specific runtime terms.
\end{enumerate}

The following chapters address those gaps directly. They are meant to make the manual more
useful both as a design reference and as a practical engineering guide.
