\section{BPMN Support And Limits}
\label{sec:SupportLimits}

\subsection{Well-supported areas}

\begin{itemize}
\item start events and explicit end completion,
\item service, user, manual, send, receive, script and business rule tasks,
\item XOR, inclusive (OR) and parallel gateway routing,
\item intermediate timer, message and signal events,
\item task and subprocess boundary timer, error and escalation handling,
\item call activities,
\item embedded subprocesses including nested subprocesses,
\item event subprocesses for message, signal, timer, error and escalation starts.
\end{itemize}

\subsection{Current semantic boundaries}

\begin{itemize}
\item cancellation propagation is pragmatic rather than globally token-precise,
\item subprocess scope handling is explicit but lighter than a full BPMN execution scope manager,
\item monitoring includes current-state indexes and coarse trend buckets, but is
  optimized for current-state inspection over long-term analytics,
\item generated runtimes prioritize inspectability over minimalism,
\item production readiness depends on the generated process implementation and
  deployment environment, not only on successful scaffolding.
\end{itemize}

\subsection{Outside current scope}

\begin{itemize}
\item full managed workflow-engine semantics,
\item compensation and advanced BPMN exception orchestration,
\item turnkey production operation without Kafka, Kubernetes, security and
  observability review,
\item long-term analytics beyond the current monitoring projections.
\end{itemize}
