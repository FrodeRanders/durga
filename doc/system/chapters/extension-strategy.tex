\section{Generated Code Extension Strategy}
\label{sec:ExtensionStrategy}

Generated code is a starting point. The BPMN model is the primary design source; generated
code is scaffolding expected to be completed and adapted.

\subsection{What is edited}

Common edits in generated projects:
\begin{itemize}
\item task business logic,
\item XOR decision conditions,
\item payload handling and validation,
\item integration calls inside service-style workers,
\item operational configuration values.
\end{itemize}

\subsection{What should remain generated}

Keep these close to the generated structure:
\begin{itemize}
\item topic and channel topology,
\item lifecycle event emission shape,
\item probe script surface,
\item subprocess and boundary wiring,
\item monitoring-facing event semantics.
\end{itemize}

\subsection{Regeneration strategy}

Preserve the overall generated runtime structure while filling in task-specific behavior.
When the BPMN model changes substantially, regenerate and re-apply focused logic changes
rather than manually rewiring the generated topology.
